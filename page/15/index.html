테마 선택

여기로 가면 헥소에서 사용할 수 있는 테마들이 있다.

마음에 드는 테마를 고르고 들어가면 문서탭에서 설치방법을 확인할 수 있다.

여기서는 next테마를 사용합니다.

테마 다운로드

여기로 이동하면 테마 설치방법이 나온다.

아주복잡해 보이지만 두가지만 진행하면 테마를 다운받을 수 있다.

  1. 로컬컴퓨터의블로그 디렉토리로 이동한다.
  2. git clone https://github.com/theme-next/hexo-theme-next themes/next

블로그 디렉토리는 hexo init할때 만들어진 위치입니다.

해당 파일들이 있는곳

두단계만 진행하면 블로그 디렉토리/theme/에 next폴더가 생긴걸 확인할 수 있다.

테마 적용

테마를 다운받고 난 후, 블로그 디렉토리에서 _config.yml을 에디터로 열어보자.

_config.yml파일의 가장 하단으로 내려가다보면 theme라는 텍스트가 보이는데 해당 부분을 우리가 바꿨던 테마의 이름으로 바꿔주어야 한다.

여기서는 next 테마를 다운받았으니
next로 수정합니다.

기존테마 지우기

Hexo theme default값은 landscape

이제 해당 테마의 파일들은 필요가 없으니 지워주고 새로운 테마의 파일들을 생성해 주어야합니다.

다음과 같이 입력

hexo clean: 테마 캐시를 삭제
hexo generate: 설정한 테마의 파일들을 생성

테마 적용 확인

위 과정을 다 했다면 hexo server를 입력하여 로컬호스트를 띄워서 확인해보도록 하자.

별 다른 설정이 없었다면
localhost:4000으로 열리며,
브라우저에서 해당주소를 입력하면 확인해 볼 수 있다.

함수,조건,반복문

함수

  • 기본적인 함수의 형태
  • 매개변수가 없는경우
  • 반환값이 없는경우
  • 매개변수와 반환값이 없는경우

매개변수의 입력타입은 무조건 입력해야한다.

다만 입력값의 타입을 도저히 모를경우에는 Any를 넣고 TypeCasting후 사용

반환값의 경우에도 마찬가지이다.

반환되는 값이 여러개 이거나 타입이 다를경우 반환타입에 Any를 넣어서 사용한다.

함수 고급

  • 기본값을 가지는 매개변수

기본값이 있는 매개변수는 매개변수 목록중 가장 뒤에 위치한다.

  • 전달인자 레이블

함수를 호출할 때 매개변수의 역할을 명확하게 하거나, 함수 사용자 입장에서 표현하고자 할 때 사용한다.

ex)

함수명이 중복되었지만 전달인자 레이블을 사용하여 다른 함수로 취급된다.

argument부분에서 to,from을 사용하여 매개변수를 전달하고 함수 내부에서 friend,me를 사용하게 된다.

  • 가변 매개변수

argument의 갯수를 알기 어려울 때 사용

가변매개변수는 함수당 하나만 가질 수 있고 목록중 가장 뒤에 위치하는것이 좋다.
