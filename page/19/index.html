git branch 연습

작성일 2019-11-12 Edited on 2019-11-13 In git Disqus:

깃 연습

git branch

  • git branch : 해당 git 폴더에서 브랜치 목록을 보여준다.

  • git branch <브랜치이름> : 해당 git 폴더에서 <브랜치이름>의 브랜치를 생성한다.

  • git checkout <이동할 브랜치 이름> : 해당 git 폴더에서 <이동할 브랜치이름>을 입력하면 브랜치가 이동한다.

  • git checkout -b <브랜치이름> : 해당 브랜치를 생성하고 이동한다.

Switched to branch <이동할 브랜치이름>이 shell출력으로 나타난다.

git branch를 다시입력하면 브랜치가 변경된걸 확인할 수 있다.

  • git branch -D <삭제할 브랜치 이름> : 해당 브랜치를 삭제

Deleted branch <브랜치이름>.이 shell출력으로 나타난다.

  • git diff <브랜치이름> : 현재 브랜치와 입력한 브랜치의 코드 차이점을 보여준다

  • git merge <브랜치이름> : 현재 브랜치에 입력한 브랜치를 합친다.

시나리오

Fork and merge 시나리오

팀장의 경우

  1. create-repo (public/init README.md)

  2. repo 주소 복사

  3. 내컴퓨터에 clone

  4. git flow init

  5. git branch

develop 브랜치 생성 확인

  1. README.md 수정

  2. add -> commit -> push(git push -u옵션)

  3. 팀원이 14.까지 끝냈다면 pull request가 생긴다.

  4. 내용을 확인하고 confirm ,reject여부를 결정

  • reject의 경우

    1. 팀원은 다시 내용을 수정해서 add -> commit -> push를 한다.
    2. github pull request에서 변경내역이 확인 가능하다.

  • confirm의 경우

  1. merge를 진행
  2. 결과 확인

팀원의 경우

  1. 팀장의 repo를 Fork

팀원 계정에 팀장의 repo를 복사하여 생긴다.

  1. 1.에서 fork한 repo의 주소를 복사

  2. 팀장의 repo에 가서 Issue 생성

Issue는 제안/신고/건의등을 하는공간

  1. Issue를 적고 Issue number 확인

  2. 내 컴퓨터에 git clone

  3. git flow init -> enter

  4. git pull origin develop

  5. git flow feature start <기능명>

  6. 기능작성

  7. add -> commit

  8. git flow feature finish <기능명>

  9. git push origin develop

  10. github에서 pull request를 날린다.

팀원의 develop -> 팀장의 develop으로 보낸다.

  1. pull request memo

테마 선택

여기로 가면 헥소에서 사용할 수 있는 테마들이 있다.

마음에 드는 테마를 고르고 들어가면 문서탭에서 설치방법을 확인할 수 있다.

여기서는 next테마를 사용합니다.

테마 다운로드

여기로 이동하면 테마 설치방법이 나온다.

아주복잡해 보이지만 두가지만 진행하면 테마를 다운받을 수 있다.

  1. 로컬컴퓨터의블로그 디렉토리로 이동한다.
  2. git clone https://github.com/theme-next/hexo-theme-next themes/next

블로그 디렉토리는 hexo init할때 만들어진 위치입니다.

해당 파일들이 있는곳

두단계만 진행하면 블로그 디렉토리/theme/에 next폴더가 생긴걸 확인할 수 있다.

테마 적용

테마를 다운받고 난 후, 블로그 디렉토리에서 _config.yml을 에디터로 열어보자.

_config.yml파일의 가장 하단으로 내려가다보면 theme라는 텍스트가 보이는데 해당 부분을 우리가 바꿨던 테마의 이름으로 바꿔주어야 한다.

여기서는 next 테마를 다운받았으니
next로 수정합니다.

기존테마 지우기

Hexo theme default값은 landscape

이제 해당 테마의 파일들은 필요가 없으니 지워주고 새로운 테마의 파일들을 생성해 주어야합니다.

다음과 같이 입력

hexo clean: 테마 캐시를 삭제
hexo generate: 설정한 테마의 파일들을 생성

테마 적용 확인

위 과정을 다 했다면 hexo server를 입력하여 로컬호스트를 띄워서 확인해보도록 하자.
