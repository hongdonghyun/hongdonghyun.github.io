깃 연습

git branch

  • git branch : 해당 git 폴더에서 브랜치 목록을 보여준다.

  • git branch <브랜치이름> : 해당 git 폴더에서 <브랜치이름>의 브랜치를 생성한다.

  • git checkout <이동할 브랜치 이름> : 해당 git 폴더에서 <이동할 브랜치이름>을 입력하면 브랜치가 이동한다.

  • git checkout -b <브랜치이름> : 해당 브랜치를 생성하고 이동한다.

Switched to branch <이동할 브랜치이름>이 shell출력으로 나타난다.

git branch를 다시입력하면 브랜치가 변경된걸 확인할 수 있다.

  • git branch -D <삭제할 브랜치 이름> : 해당 브랜치를 삭제

Deleted branch <브랜치이름>.이 shell출력으로 나타난다.

  • git diff <브랜치이름> : 현재 브랜치와 입력한 브랜치의 코드 차이점을 보여준다

  • git merge <브랜치이름> : 현재 브랜치에 입력한 브랜치를 합친다.

시나리오

Fork and merge 시나리오

팀장의 경우

  1. create-repo (public/init README.md)

  2. repo 주소 복사

  3. 내컴퓨터에 clone

  4. git flow init

  5. git branch

develop 브랜치 생성 확인

  1. README.md 수정

  2. add -> commit -> push(git push -u옵션)

  3. 팀원이 14.까지 끝냈다면 pull request가 생긴다.

  4. 내용을 확인하고 confirm ,reject여부를 결정

  • reject의 경우

    1. 팀원은 다시 내용을 수정해서 add -> commit -> push를 한다.
    2. github pull request에서 변경내역이 확인 가능하다.

  • confirm의 경우

  1. merge를 진행
  2. 결과 확인

팀원의 경우

  1. 팀장의 repo를 Fork

팀원 계정에 팀장의 repo를 복사하여 생긴다.

  1. 1.에서 fork한 repo의 주소를 복사

  2. 팀장의 repo에 가서 Issue 생성

Issue는 제안/신고/건의등을 하는공간

  1. Issue를 적고 Issue number 확인

  2. 내 컴퓨터에 git clone

  3. git flow init -> enter

  4. git pull origin develop

  5. git flow feature start <기능명>

  6. 기능작성

  7. add -> commit

  8. git flow feature finish <기능명>

  9. git push origin develop

  10. github에서 pull request를 날린다.

팀원의 develop -> 팀장의 develop으로 보낸다.

  1. pull request memo
