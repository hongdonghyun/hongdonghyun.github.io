Hexo로 Github Page 만들기

Hexo

Node.js기반의 정적페이지 생성기

사용자가 급격하게 상승중

오픈소스는 사용자가 많은게 짱임

시작하기전

github페이지를 만들어야 하기 때문에 github계정은 필수이며,

간단한 터미널 명령어는 사용할 수 있어야한다.

또한 깃허브페이지는 계정당 하나만 생성할 수 있다.

레포이름에 계정명이 들어가야하기 때문

  1. git
  2. github 계정
  3. 터미널
  4. markdown 문법
  5. vi 혹은 emacs같은 익숙한 에디터

해당 게시물에서는 MacOS기준으로 설명한다.

설치

레포생성

cd명령어로 projects까지 이동 후 hexo init 폴더명을 지정했습니다.

포스팅 작성

포스팅 생성

터미널에서 다음과 같이 입력하자.

hexo new 포스팅제목
해당 커맨드를 입력하면 init에서 지정한 폴더명/source/_posts/에 포스팅제목.md가 생긴걸 확인할 수 있다.

포스팅 확인

cd 명령어로init에서 지정한 폴더명/source/_posts/로 이동한 후 파일이 존재하는지 ls로 확인해보자.

저는 first-posting이라는 이름으로 생성했습니다.

hello-world.md는 초기에 생성해주는 파일이에요.

해당 파일을 에디터로 열어보게 되면

다음과 같은 부분이 나타나는데
포스팅을 등록할때 해당부분을 채워주면된다.

배포

이제 hexo도 설치했고, 로컬컴퓨터에서 동작도 확인했고, 포스팅을 작성하는법도 알았다.

남은건 이제 인터넷 세상에 블로그를 공개하는일만 남았다.

config

배포를 하기위해선 아까 만들어 두었던 github page와 연동을 해줘야한다.

블로그폴더 최상단으로 이동하자.

여기

여기서 _config.yml을 수정해보도록 한다.

수정하기에 앞서 여기를 한번 보고가면 편하다.

가장 기본적인 부분만 수정해보도록 하자.

저는 이런식으로 수정했습니다.

deploy config

_config.yml파일 제일밑으로가면
#Deployment부분이 있다.

해당부분을 git을 사용해 githubpage를 띄우도록 수정한다.

repo에 들어갈 주소는 포스팅 처음에 시작했던 레포지토리에서 clone or download를 누르면 나오는 주소를 복사해서 넣어주면됩니다.

정적파일생성

hexo generate를 입력하면 정적파일들을 만들어낸다.

배포

배포하기전 하나의 과정이 더 남았다.
hexo-deployer-git라는 플러그인을 설치해 주어야만 진정으로 배포가 가능해진다.

배포결과 확인

배포가 다되었다면 깃허브에서 뭔가 주루룩하고 올라와있다.

배포를 하고 1분만 기다렸다가 자신의 사이트로 접속해보자.

github이 바빠서 바로바로 업데이트를 못해주는 경우가 있기때문

위와 같은 사이트가 나온다면 완벽하게 블로그를 생성해낸것이다.
